# references/

이 폴더는 블로그 글에서 허용되는 자료 출처 기준을 정리하는 자리입니다.

## 이 파일은 사용자가 직접 관리합니다

이 파일(`references/README.md`)의 출처 정책 — 허용 출처, 사용하지 않는 출처, 우선순위 — 은 기본적으로 사용자가 직접 수정하고 항목을 추가하는 것을 전제로 합니다. Claude Code와 Codex는 사용자가 명시적으로 수정을 요청하지 않는 한 이 파일을 고치거나 항목을 추가하지 않습니다.

이 파일은 저장소 전체의 출처 정책을 결정하는 기준 문서이므로, 그 변경은 사람의 판단으로만 이루어져야 합니다. AI가 할 수 있는 일은 다음까지입니다.

- 정책에 손볼 곳이 보이면 사용자에게 보고하고 수정 문구를 제안합니다.
- 사용자가 명시적으로 요청하면, 요청한 범위 안에서만 이 파일을 편집합니다.
- 이 파일이 바뀐 뒤에는, 그 변경을 따라가야 하는 요약 문서(`README.md`, `CODEX.md`, `docs/`, `.claude/`, `templates/`)를 갱신합니다.

명시적 요청이 없으면 이 파일은 읽기만 하고 수정하지 않습니다.

## 우선순위

출처 정책의 최우선 기준은 이 파일입니다. 다른 문서에 출처 허용 범위가 요약되어 있더라도, 충돌이 생기면 이 파일의 내용이 우선합니다. 출처 정책을 바꿀 때는 이 파일을 먼저 수정하고, `README.md`, `CODEX.md`, `docs/`, `.claude/`, `templates/` 문서는 그 변경을 따라가도록 갱신합니다.

## 직접적인 출처 이름과 URL은 이 파일에만 적습니다

구체적인 출처 이름, 도메인, URL은 이 파일에만 둡니다. `README.md`, `CODEX.md`, `docs/`, `.claude/`, `templates/` 등 다른 문서에는 직접적인 출처 이름이나 URL을 적지 않고, "허용 출처는 `references/README.md`를 따른다"는 식으로 이 파일을 가리키기만 합니다. 출처 기준이 한곳에 모여 있어야 정책이 바뀔 때 갱신 지점이 분명해지고, 문서마다 출처 설명이 어긋나는 일이 생기지 않습니다.

예외는 글 자체입니다. `posts/`와 `drafts/`의 글 frontmatter `sources`에는 그 글이 실제로 인용한 자료의 제목과 URL을 적습니다. 그것은 출처 정책이 아니라 인용 기록이기 때문입니다.

## 허용 출처

1. Microsoft 공식 문서 — `learn.microsoft.com`의 공식 문서를 사용합니다.
2. 법령 및 공공기관 가이드라인 — `law.go.kr`, `*.go.kr` 등 공공기관 공식 자료를 사용합니다.
3. 보안 솔루션 업체 — 업체의 공식 페이지나 백서만 사용합니다.

## 사용하지 않는 출처

블로그, 커뮤니티, 언론 기사, 유튜브, 위키는 본문 출처로 사용하지 않습니다. Microsoft 자료라도 공식 문서가 아닌 Microsoft 블로그는 발행 출처로 쓰지 않습니다.

자료 조사 과정에서 보조적으로 읽을 수는 있지만, frontmatter의 `sources`에는 위의 허용 출처만 기록합니다.
