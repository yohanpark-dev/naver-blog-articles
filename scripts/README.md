# scripts/

자동화 스크립트가 들어가는 자리입니다.

처음에는 비어 있는 것이 정상입니다. 글이 충분히 쌓이기 전까지는 Claude Code의 슬래시 명령어로 같은 일을 더 유연하게 할 수 있기 때문입니다. 글이 50편 이상 쌓이고 같은 작업이 반복적으로 일어나면 그때 스크립트로 만드는 것이 효율적입니다.

먼저 만들 만한 스크립트는 다음과 같은 것들이 있습니다.

`validate.js`는 발행 전 검사를 일괄로 돌리는 스크립트입니다. Claude Code의 `/publish-check`와 같은 일을 하지만, 명령행에서 즉시 실행할 수 있습니다. CI 파이프라인에 붙이면 git push 시점에 자동으로 검사를 돌릴 수 있습니다.

`find-related.js`는 모든 글의 frontmatter를 훑어서 `topic_keywords`로 매칭되는 관계를 그래프로 출력하는 스크립트입니다. 시간이 지나 글이 쌓이면 어떤 주제 그룹이 형성되었는지를 시각적으로 보는 데 도움이 됩니다.

`stats.js`는 발행 통계를 보여주는 스크립트입니다. 월별 발행 수, 카테고리별 분포, `volatility` 별 점검 대상 글 수 같은 정보를 한 번에 확인할 수 있습니다.

지금은 이 폴더에 아무것도 없어도 시스템이 작동합니다. 필요해질 때 만듭니다.
